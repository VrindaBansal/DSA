<Intuition>
The same model can cost cents or dollars, answer in 200ms or 20s, depending
entirely on how it's *served*. Two phases govern everything: **prefill** reads
your whole prompt in parallel to produce the first token (this sets
time-to-first-token — long prompts hurt here), and **decode** generates tokens
one at a time using the KV cache (this sets throughput — long outputs hurt
here). The big serving win, **continuous batching**, keeps the GPU saturated by
swapping requests in and out per token instead of waiting for a fixed batch.
And the real capacity limit usually isn't compute — it's the KV cache's memory,
which grows with every token of context. Understand this and the token bill,
the latency numbers, and "why does long context cost so much?" all stop being
mysterious.
</Intuition>
