<Intuition>
A raw model can only produce text. Tool use is what lets it *reach into the
world* — look something up, do real math, query a database, send a request.
The mechanism is more mundane than it sounds and worth burning into memory: the
model never runs anything. It emits a **structured request** — a tool name and
JSON arguments — and *your* code executes the tool, then hands the result back
for the model to continue. It's a loop you orchestrate, which is exactly where
you enforce validation and authorization. Tool descriptions are prompt
engineering (the model picks tools by their names and docstrings), and every
tool that touches money or data is an injection target. This is the bridge from
"chatbot" to "software that acts."
</Intuition>
