<Intuition>
You already run a stack all day: every function call pushes a frame, every
return pops one, and a stack trace is just the stack printed mid-flight.
LIFO is the natural shape of **nesting** — the most recently opened thing
must close first, whether it's a bracket, an HTML tag, or a database
transaction. The sleeper hit of this module is the *monotonic* stack: keep
the stack sorted by popping violators, and a whole family of "next greater
element" problems collapses from O(n²) to O(n).
</Intuition>
