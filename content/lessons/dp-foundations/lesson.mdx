<Intuition>
Dynamic programming has a reputation it doesn't deserve. It is one move,
applied relentlessly: **your recursion keeps re-solving the same
subproblems — stop doing that.** Cache them (memoization) or fill them in
order (tabulation); either way the exponential tree from Module 1 flattens
into a table. The visual fills the *same* edit-distance table both ways —
top-down's sparse, need-driven fill against bottom-up's dense sweep. Same
cells, two orders: once you see that, DP is bookkeeping, not magic. The
arrows in the table are literally your diff tool's output.
</Intuition>
