<Intuition>
A BST is binary search made into a place you can live: left is smaller,
right is bigger, so every lookup walks one root-to-leaf path. All costs are
O(height) — which is the whole story, because nothing forces height to be
log n. Feed the tree sorted input below and watch it collapse into a
linked list. That degeneration — not aesthetics — is why AVL, red-black
trees, and the B-trees under every SQL index exist: they are machines for
keeping height honest.
</Intuition>
